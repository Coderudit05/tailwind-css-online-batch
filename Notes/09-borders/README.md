# Tailwind CSS - Borders (Project 09)

## Concept First: Why Border Utilities Matter

Borders define boundaries and improve clarity between UI elements. In Tailwind, borders are controlled through width, color, side-specific utilities, radius, and outline.

Why important:

- Better visual grouping of content.
- Clear form states and focus indication.
- Cleaner card and component styling.

---

## Lesson-wise Notes

### 1. Border Width and Color

Examples:

- `border`
- `border-2 border-blue-500`
- `border-4 border-red-500`

---

### 2. Side-Specific Borders

Examples:

- `border-x-4`
- `border-y-4`
- `border-l-4`
- `border-r-4`

---

### 3. Border Radius

Examples:

- `rounded`
- `rounded-lg`
- `rounded-xl`
- `rounded-2xl`
- `rounded-full`

---

### 4. Side-Specific Radius

Examples:

- `rounded-t-xl`
- `rounded-b-xl`

---

### 5. Outline Utilities

Examples:

- `outline-1`
- `outline-2`
- `outline-4`
- `outline-offset-2`

Outline is outside the border and commonly used for focus.

---

## Quick Revision Summary

- Border width and color create emphasis and separation.
- Border sides help build custom component style.
- Radius controls corner softness.
- Outline improves accessibility and focus visibility.
